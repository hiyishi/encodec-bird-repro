# EnCodec Bird Bioacoustic Continuation Pack

This repository is a minimal continuation and reproduction pack for the EnCodec-centered bioacoustic compression experiments in Bio-FARGAN / PS-CX A133-A140.1.

It is meant for restarting the work on another machine with another local dataset, while keeping enough context to avoid repeating already-consumed routes.

It does not include our data, run outputs, or internal project helpers. Use your own WAV files through a simple manifest.

## If You Are Resuming This Project

Start here:

```text
docs/NEXT_START_HERE.md
```

That file gives the recommended first-day workflow, the no-promote routes to avoid restarting from, and the decision tree for whether to continue EnCodec, switch codec family, or frame the work as a benchmark / negative-oracle study.

## What This Reproduces

The pack covers the decision-critical EnCodec experiments:

```text
01_build_anchors.py
  Rebuild EnCodec K=2/K=4/K=8 anchors: approximately 1.5/3/6 kbps.

02_event_masked_rvq_oracle.py
  A137-style frozen EnCodec event-masked RVQ depth allocation.

03_minimal_finetune.py
  A138-style neutral or event-weighted minimal EnCodec fine-tuning sanity check.

04_detail_oracles.py
  A139/A140-style exact-detail upper bounds:
    E0.75 + event residual at 1.5 kbps total
    E1.5 + 0.10/0.25/0.50 kbps marginal detail UB
```

These scripts are intended to reproduce the experimental logic and restart the research loop, not our exact dataset-specific verdicts.

## Manifest

Prepare a CSV like `examples/manifest.example.csv`.

Required columns:

```text
clip_id,wav_path
```

Optional columns:

```text
split,species,group
```

Relative `wav_path` values are resolved relative to the manifest file.

## Minimal Run

```powershell
python scripts\01_build_anchors.py --manifest examples\manifest.example.csv --out_dir runs\anchors --clean
python scripts\02_event_masked_rvq_oracle.py --manifest examples\manifest.example.csv --out_dir runs\a137_masked_rvq --clean
python scripts\04_detail_oracles.py --manifest examples\manifest.example.csv --out_dir runs\detail_oracles --mode both --clean
```

Optional neutral fine-tune smoke:

```powershell
python scripts\03_minimal_finetune.py --manifest examples\manifest.example.csv --out_dir runs\minimal_ft --steps 20 --clean
```

Optional event-weighted loss smoke:

```powershell
python scripts\03_minimal_finetune.py --manifest examples\manifest.example.csv --out_dir runs\eventloss_ft --loss_mode event --steps 20 --clean
```

Use more clips and more steps only after the small smoke produces a valid review bundle.

## Listening Discipline

Objective metrics are only routing diagnostics. They cannot promote a codec route.

Promotion requires listening within the same generated review bundle:

```text
same source clips
same current EnCodec decode path
same output sample rate
same route ordering
```

Do not compare against old WAVs generated by a different decode path unless the purpose is historical context only.

## Important Boundaries

The exact-detail routes in `04_detail_oracles.py` are illegal upper bounds:

```text
They inject source residual into selected event frames.
They are not deployable bitstreams.
They only test whether a detail-token design would have theoretical listening value.
```

If an exact-detail upper bound does not audibly beat the EnCodec anchor, a legal compressed version is unlikely to be worth building.

## Suggested Verdict Sheet

For every clip and route family, record:

```text
clip_id
route_pair
winner: anchor / candidate / tie
artifact_type: none / metallic / watery / muffled / noisy / harsh / dull
event_clarity: better / tie / worse
background_damage: yes / no
promotion_note
```

## Relation To The Bio-FARGAN Results

Our internal verdict was:

```text
Frozen RVQ event allocation:
  no clear listening improvement.

Small neutral/event-loss EnCodec FT:
  no stable perceptual specialization.

Same-rate E0.75 + exact event residual UB:
  beats E0.75 variants but loses E1.5.

E1.5 + 0.10/0.25/0.50 kbps exact-detail UB:
  no clear audible difference from E1.5.
```

Treat this pack as a way to test whether that conclusion holds on a new dataset, and as a launchpad for the next branch if it does not.
