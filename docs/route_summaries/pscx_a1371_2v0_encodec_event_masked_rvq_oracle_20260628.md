A137.2v0 falsifies post-hoc EnCodec RVQ depth allocation. The implementation correctly performs prefix-monotone, per-layer latent masking inside frozen EnCodec RVQ latents. However, `event4` and `event8` do not produce clear listening improvement over fixed EnCodec anchors. Therefore, existing frozen EnCodec RVQ layers are not locally reusable enough to improve bird-event quality by inference-time depth allocation.

This closes the post-hoc allocation branch. It does not close EnCodec-based specialization in general; it redirects the next question to training-time specialization.

## Technical Contract

- Full EnCodec codes are loaded as `[B, K, T]` from existing A1347 6 kbps packages.
- Each RVQ layer is decoded independently with `model.quantizer.vq.layers[k].decode(codes[:, k, :])`.
- Masking is applied in latent space: `z_q(t) = sum_k z_q_k(t) * mask[k,t]`.
- Prefix-monotone depth is enforced by `active_k[t]`; allowed examples include K=2 background and K=4/8 event frames.
- Inactive codebooks are never replaced with code index 0.
- Decoder is frozen EnCodec 24 kHz; no decoder fine-tuning is performed.

## Implementation Audit

After listening failed, the oracle implementation was checked to avoid mistaking a bug for a route failure.

- `E6` package prefixes are identical to the separately encoded `E1.5` and `E3` package codes on the 15-case subset.
- Per-layer latent summation is equivalent to `model.quantizer.decode` for fixed depths: `K=2` and `K=4` are exact; `K=8` differs only at floating-point noise level.
- A137 fixed WAVs replay the same current package decode path used by `event4` and `event8`.
- `event4` and `event8` are not no-ops: waveform deltas are concentrated in event windows, but the audible result is not a useful quality improvement.

This supports the verdict: the route failed because frozen high-rate EnCodec layers are not locally reusable in the desired way, not because the mask implementation failed to activate.
